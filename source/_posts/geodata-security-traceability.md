## 问题

空间数据（矢量、栅格）一旦分发出去，就脱离了你的控制。如果数据被泄露或非法传播，你怎么知道是谁泄露的？

传统的做法是在数据里加一个标识符（水印），但这面临几个挑战：

1. 水印要**不可见** — 不能影响数据的正常使用
2. 水印要**不可移除** — 攻击者不能轻易去掉
3. 水印要**可验证** — 任何人都能验证水印的存在和归属
4. 水印要**防伪造** — 不能被篡改成别人的标识

GeoData Security System 就是为了解决这个问题。

## 系统设计

### 整体流程

```
用户申请数据 → 双级审批 → 水印嵌入 → 安全分发 → 溯源验证
```

1. **申请** — 用户提交数据使用申请
2. **审批** — 两级管理员审批（双重保障）
3. **水印** — 系统自动在数据中嵌入 QR 码水印
4. **分发** — 带水印的数据发放给用户
5. **溯源** — 发现泄露时，提取水印定位责任人

### QR 码水印系统

这是系统的核心。水印不是简单的文字标记，而是 QR 码——它本身携带信息，且可以通过二维码解码直接读取。

**矢量数据（SHP）**：在几何坐标中嵌入微小偏移，偏移模式编码了 QR 码信息。肉眼看不出区别，但算法可以提取。

**栅格数据（GeoTIFF）**：支持 3 种算法：

| 算法 | 原理 | 特点 |
|------|------|------|
| LSB | 最低有效位替换 | 简单，容量大，鲁棒性一般 |
| DWT | 离散小波变换 | 鲁棒性强，抗压缩 |
| 直方图偏移 | 像素值分布调整 | 无损，可逆 |

所有栅格算法都保留了 GeoTIFF 的 CRS（坐标参考系统）信息，确保水印不影响地理定位。

### HMAC-SHA256 签名

水印内容用 HMAC-SHA256 签名。这意味着：

- 水印内容防篡改 — 改了内容，签名就对不上
- 只有服务端有密钥 — 外部无法伪造有效水印
- 可验证 — 任何人都能用公钥验证签名

## 角色与权限

系统有三种角色：

- **普通员工** — 申请数据、查看自己的申请
- **管理员（adm1/adm2/adm3）** — 审批申请，不同级别对应不同审批阶段
- **超级管理员** — 管理用户、系统配置

### 双级审批

不是一个人说了算。申请需要两级管理员都通过才能生效。这防止单点腐败——一个管理员放水没用，另一个会拦住。

### 民主召回机制

如果发现数据被滥用，管理员可以发起召回投票。超过 50% 的管理员反对，数据就会被召回。

这是一种制度设计——技术手段（水印）和制度手段（投票召回）双保险。

### 管理员晋升

员工可以申请成为管理员，需要 66% 的现有管理员同意。这保证了管理团队的质量。

## 技术栈

| 层 | 技术 | 用途 |
|----|------|------|
| 前端 | Vue 3 + Element Plus | 企业级 UI |
| 后端 | Flask + Flask-RESTful | REST API |
| 认证 | JWT（Access + Refresh） | 身份验证 |
| 数据库 | MySQL + PostgreSQL/PostGIS | 业务 + 空间数据 |
| 缓存 | Redis | 热数据缓存 |
| 实时 | Socket.IO | 申请状态推送 |
| 地图 | Leaflet | 空间数据可视化 |
| 监控 | Prometheus + Grafana | 指标采集 + 仪表盘 |
| 日志 | Loki + Promtail | 日志聚合 |
| 部署 | Docker Compose | 7 个服务容器化 |

### 双数据库架构

MySQL 存业务数据（用户、申请、日志），PostgreSQL + PostGIS 存空间数据。为什么要分两个？

- PostGIS 是空间数据的事实标准，矢量数据的查询（相交、包含、距离）需要空间索引
- MySQL 的业务生态更成熟，用户管理、权限控制用 MySQL 更方便
- 分离关注点，空间数据的备份和业务数据的备份策略不同

### 监控体系

Prometheus 采集 11 个指标面板（请求速率、延迟、错误率、业务 KPI），Grafana 可视化 + 告警。

告警规则：
- 错误率超过阈值
- 延迟超过阈值
- 数据库连接异常
- 缓存命中率过低

Loki + Promtail 做日志聚合，可以在 Grafana 里直接查看日志，不用 SSH 到服务器。

## 安全设计

- **JWT 双令牌** — Access Token 短有效期（15 分钟），Refresh Token 长有效期（7 天），自动续期
- **按用户限流** — 不是按 IP 限流，而是按 JWT 身份限流。一个用户不能因为换 IP 就绕过限制
- **操作审计日志** — 所有关键操作都有记录，可按用户、操作类型、时间范围筛选
- **i18n** — 中英文双语支持，49 个视图、1200+ 翻译键

## 总结

做 GeoData Security System 最大的体会是：**数据安全不只是技术问题，更是制度问题。** 技术手段（水印、签名）解决"能不能追踪"的问题，制度手段（双级审批、民主召回）解决"该不该追踪"的问题。两者缺一不可。
