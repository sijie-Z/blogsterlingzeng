## 为什么做这个

量化交易的工具很多——聚宽、米筐、优矿，开箱即用。但这些平台有两个问题：

1. **黑盒** — 你不知道回测引擎内部怎么处理的，数据有没有前视偏差
2. **不可控** — 想加一个自定义因子、改一下成本模型，要么不支持，要么很麻烦

我想从零搭一个，目的不是实盘赚钱（那需要完全不同的工程量），而是**搞清楚量化研究的全链路**——从数据到因子到回测，每一步都自己实现一遍。

## 系统架构

整个平台是一条完整的流水线：

```
数据层 → 因子引擎 → Alpha 模型 → 组合优化 → 回测引擎 → 风险管理 → 报告
```

每个模块都是独立的，可以单独替换或扩展。

### 数据层

数据是量化的基础。我做了两个 DataProvider：

**合成数据生成器** — 生成 500 只虚拟 A 股、5 年的日线数据。好处是不需要 API key，跑测试快，而且数据里内置了 alpha 结构，方便验证因子是否有效。

```python
# 合成数据里埋了 alpha 信号
# 如果你的因子能挖出来，说明实现是对的
```

**Tushare 实盘数据** — 接入真实 A 股数据。但实盘数据有大量坑：停牌、涨跌停、ST 股票、前复权、幸存者偏差……我在 `data/ASHARE_PITFALLS.md` 里整理了 10 个 A 股特有的陷阱，每个都做了显式处理。

### 因子引擎

做了 15 个因子：

- **10 个技术因子** — 动量、波动率、换手率、成交量比率等
- **5 个基本面因子** — PE、PB、ROE、营收增长、净利润增长

每个因子继承 `BaseFactor`，注册到 `FactorRegistry`，支持热插拔。

因子处理流程：缩尾（去极值）→ 标准化 → 中性化（行业、市值）。

### 因子评估

光算出因子不够，还得评估它有没有用。我实现了完整的因子评估体系：

| 指标 | 含义 |
|------|------|
| Rank IC | 因子值与未来收益的秩相关 |
| Pearson IC | 皮尔逊相关 |
| ICIR | IC 的信息比率（IC 均值 / IC 标准差） |
| 分位数收益 | 按因子值分 5 组，看各组收益差异 |
| IC 衰减 | IC 随持有期变化的曲线 |

ICIR > 0.5 的因子才值得用。这个筛选标准帮我砍掉了一大半"看起来有用但其实没用"的因子。

### Alpha 模型

因子合成的方法有三种：

- **等权** — 最简单，作为基准
- **IC 加权** — IC 高的因子权重高
- **ICIR 加权** — ICIR 高的因子权重高（更稳定）

### 组合优化

从因子信号到实际持仓，需要优化器：

- **等权基准** — 不优化，作为对照
- **均值方差** — Markowitz，用 cvxpy 求解
- **风险平价** — 每个资产贡献相等的风险

支持行业约束和换手率约束。

### 回测引擎

月频向量化回测，完整的 A 股成本模型：

```python
# A 股交易成本
commission = 0.0003   # 万三佣金（买卖各收）
stamp_tax = 0.0005    # 万五印花税（只卖方收）
slippage = 0.001      # 千一滑点
```

### 风险管理

- VaR（历史模拟法、参数法）
- CVaR（条件在险价值）
- 压力测试（2015 股灾、2020 疫情等历史场景）
- 行业暴露分析

## 踩过的坑

### 1. 前视偏差

最常见的回测陷阱。比如用当天的收盘价算因子，用当天的收盘价下单——这在实盘中不可能做到，因为收盘后你才能拿到数据。

解决方案：所有因子计算用 T-1 数据，交易用 T+1 开盘价。

### 2. 幸存者偏差

A 股有退市股票。如果只用当前存续的股票回测，相当于默认你知道哪些股票不会退市——这是未来信息。

解决方案：用历史全量股票池，包括已退市的。

### 3. 停牌处理

停牌股票不能交易，但很多量化框架直接跳过了。这会导致回测结果虚高。

解决方案：停牌日标记为不可交易，组合优化时排除。

## 成果

- 105 个单元测试全部通过
- 合成数据端到端约 3 分钟，Tushare 实盘约 5 分钟
- 完整的因子评估报告（IC、ICIR、分位数收益、相关性矩阵）
- 净值曲线、回撤、滚动 Sharpe、月度热力图等可视化报告
- LLM 情感因子（可选，接入 OpenAI）

## 总结

做这个平台最大的收获是理解了量化研究的本质：**不是找一个神奇的公式能预测涨跌，而是一套严谨的统计方法论。** 因子有没有用，不是看它"听起来有没有道理"，而是看 IC/ICIR 的统计显著性。回测结果好不好，不是看收益率高不高，而是看有没有犯前视偏差、幸存者偏差这些基本错误。

量化交易最难的不是写代码，是保持纪律。
